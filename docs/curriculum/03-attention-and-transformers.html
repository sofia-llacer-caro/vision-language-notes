When it comes to the Query, Key and Value, from the point of view of a given word, Query is asking “which words are relevant to me?”; Key is saying “here is what I contain”; and Value is deciding which information gets put forward.

On top of that, self-attention helps clear out some possible semantinc misunderstandigs which can only be resolved with context and neighboring words.
  3. Multi-head attention is a mechanism in which attention is computed multiple times in parallel (in the case of GPT, for instance, it’s 96). This is done as to capture slightly different contextual aspects of language in each of them.
  4. Positional encoding, which works by adding a fixed vector to each token embedding before processing, encoding its position in the sequence. Notes

Transformers for Text

The goal of a transformer is to take a piece of text and produce a probability distribution over the next word. To do this, words are first converted into embeddings, which are vector representations of semantic meaning. A key property of transformers is that these embeddings are not fixed, but instead they are updated based on context through the attention mechanism.

For instance, the word “mole” has a generic context-free embedding at first. Depending on surrounding words, attention can shift that embedding toward “animal”, “unit of measurement”, or “spy”, encoding richer meaning as context accumulates.

Attention: Query, Key, and Value

The attention mechanism works through three parameter matrices: Query, Key, and Value.

Taking a simple example of [adjective][noun]: the noun produces a query vector asking “which words are relevant to me?”, and each adjective produces a key vector answering “here is what I contain.” The dot product between queries and keys produces an attention pattern: a matrix of relevance scores between every pair of tokens. High dot product means high relevance (the adjective attends strongly to the noun); low dot product means low relevance (a generic word like “the” attends weakly to most things).

Since these dot products can range from \(-\infty\) to \(+\infty\), they are normalised through a softmax to produce values between 0 and 1.

The value vector then determines what information actually gets passed forward. Each word’s embedding is updated by a weighted sum of value vectors, where the weights come from the attention pattern. This produces a context-enriched embedding in place of the original context-free one.

The attention pattern matrix scales with the square of the context window, which becomes a significant computational bottleneck in large language models and is an active area of research.

Masking

During training, future tokens must not be visible to the model (this would give away the answer). This is handled by masking: setting future positions in the attention pattern to \(-\infty\) before applying softmax, which drives their weight to zero.

Multi-Head Attention and Architecture

In practice, attention is computed in parallel across many heads simultaneously (96 in GPT-3). Each head can be thought of as tracking a different type of contextual relationship. The value matrix in each head is factorised into two smaller matrices for computational efficiency, reducing parameters from roughly 150 million to around 6 million per head.

The overall architecture of a GPT alternates between attention layers and multilayer perceptron (MLP) layers, repeated across many blocks. Attention handles communication between tokens; MLPs process each token independently. Deeper layers encode increasingly abstract and nuanced meaning.

GPT-3 has 175 billion parameters in total, of which around 57 billion come from attention, the majority of which come from the MLP layers.

Self-Attention vs Cross-Attention

The attention described above is self-attention: tokens in the same sequence attend to each other. This is what GPT-style models use. Cross-attention, used in tasks like translation, allows tokens from one sequence (e.g. English) to attend to tokens from another (e.g. French).

Why Transformers Replaced RNNs

Before transformers, sequential text was handled by recurrent neural networks (RNNs). RNNs processed text token by token in order, which made them inherently sequential and difficult to parallelise. They also struggled with long texts, losing information from early tokens by the time they reached later ones.

Transformers solve both problems. By processing all tokens simultaneously and using positional encoding to preserve order information, they are highly parallelisable and scale well with both sequence length and model size.

The two core innovations of transformers are:

  • Positional encoding: word order is stored in the data itself rather than enforced by sequential processing.
  • Self-attention: context is captured by allowing every token to directly attend to every other token, regardless of distance.

Vision Transformers (ViT)

The core architecture of a Vision Transformer is identical to that of a text transformer. The difference is in how the input is represented.

An image is divided into fixed-size patches (typically 16x16 pixels). Each patch is flattened and linearly projected into an embedding vector, taking the role of a token. From that point, the architecture proceeds exactly as in the text case: positional encodings are added, and the sequence of patch embeddings is passed through alternating self-attention and MLP layers.

When ViT model names include descriptors like “Large” or “Huge”, these refer to the number of layers and parameters in the architecture. The number (e.g. ViT-16 or ViT-14) refers to the patch size in pixels.
