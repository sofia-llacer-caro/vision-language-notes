My answers

  1. A convolutional layer extracts local features from an image by sliding a small filter across the spatial dimensions and computing a dot product at each position. Each filter learns to detect one specific pattern (an edge, a texture, a colour gradient). Stacking multiple convolutional layers produces increasing abstraction: early layers detect simple features, deeper layers detect combinations of those features into increasingly complex patterns.
  2. Translation equivariance means that if a feature shifts position in the input image, its corresponding activation in the feature map shifts by the same amount. This is a useful assumption for images because objects can appear anywhere in the frame: a cat in the top left and a cat in the bottom right are still cats, and the filter should detect both without needing to be retrained for each location.
  3. As you go deeper in a CNN, the representation gains abstraction and depth: more filters track more distinct and complex features simultaneously. It loses spatial resolution, as width and height shrink through pooling. Fine-grained spatial detail is traded for richer, more abstract feature representations.
  4. Residual connections solve the vanishing gradient problem. As networks get deeper, the gradient signal shrinks as it travels backward through many layers during backpropagation, causing early layers to stop learning effectively. Skip connections give gradients a direct path backward by adding the input of a layer directly to its output, bypassing the transformation. This allows very deep networks to train effectively without degradation in performance.

Notes

Definition & Core Elements

CNNs have the same key components of the neural networks we saw before but, in this case, their loss function can be for instance an SVM or softmax and, most importantly, they assume all inputs are images. In particular, regular neural networks do not scale well to images, which is why we can have this subset of neural networks as a specialised subtype particular to the problematics commonly found in images.

CNNs have three types of layers: Convolutional Layer, Pooling Layer and Fully-Connected Layer (add ReLU as an elementwise transformation). Stacking these results in the full architecture of a CNN. As the image moves through the network, width and height shrink while depth grows, encoding increasingly abstract features. The four main layer types are:

  • CONV (Convolutional Layer). The core of the network. A set of small learned filters slides across the spatial dimensions of the input. At each position, it computes a dot product between the filter weights and the local patch of the image. Since weights can be any real number (positive or negative), the output can also be negative or positive. Each filter produces one 2D activation map, and stacking all of them gives the new depth dimension. Early filters learn to detect edges and textures; deeper filters detect increasingly abstract patterns.

  • ReLU (Rectified Linear Unit). Applied elementwise immediately after convolution. The function is simply \(f(x) = \max(0, x)\): positive values pass through unchanged, negative values are set to zero. A negative activation means the filter did not detect its feature at that location, so ReLU converts it to “nothing detected.” Without this nonlinearity, stacking layers would collapse into a single linear transformation and the network would lose expressive power.

  • POOL (Pooling Layer). Downsamples the spatial dimensions. The most common version is max pooling: a 2x2 window slides across each activation map and keeps only the maximum value in each patch, discarding the rest. This halves the width and height while preserving depth. Other examples are average pooling, for instance. The intuition is coarse-graining: integrating out fine-grained spatial detail to keep only the dominant signal, reducing computation and making the representation more compact.

  • FC (Fully-Connected Layer). At the end of the network, once the spatial dimensions have been reduced to a small size, the volume is flattened and passed through one or more fully-connected layers, exactly as in a standard neural network. The final FC layer outputs one score per class, and the class with the highest score is the prediction.

The full pipeline for a simple CNN is therefore: detect local features (CONV), introduce nonlinearity (ReLU), (repeat as many times as necessary), reduce spatial resolution (POOL), repeat until compact, then classify (FC).

Architecture Case Study

A particular case of a relevant CNN architecture is ResNet. In general, more layers tends to work better. However, one problem which arises with this is that if you just naively increase the number of layers, you end up having a higher training error. In ResNets, however, the training and testing error have a consistent improvement as you increase the number of layers.

In a resnet, there are skip connections. What that means is that, instead of computing a whole layer again, you’re just computing an addition between layer and layer without getting rid of the original layer again and again. Essentially, if you have an identity matrix, you’re just computing a changing \(\delta\) on top of it, instead of computing the whole changing matrix again and again. This makes it easier to compute with and optimise.
